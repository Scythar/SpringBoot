Factory Pattern

I need an object but don't want to specify the exact class"
Centralizes object creation logic. The client just asks for an object — Factory decides which one to create.
use when Object creation logic is **complex or repetitive**

Example:
⭐Interface & Implementations:
public interface Notification {
    void send();
}
public class EmailNotification implements Notification {
    public void send() {
        System.out.println("Sending Email");
    }
}
public class SMSNotification implements Notification {
    public void send() {
        System.out.println("Sending SMS");
    }
}

⭐Factory class:

public class NotificationFactory {

    public static Notification createNotification(String type) {
        if(type.equals("email"))
            return new EmailNotification();
        else
            return new SMSNotification();
    }
}

⭐Usage:

public class Main {

    public static void main(String[] args) {

        Notification notification =
                NotificationFactory.createNotification("email");

        notification.send();
    }
}
